## 官方训练流程理解

```text

```

关键 checkpoint：

```text
WORLD_MODEL_POSTTRAIN
UNIVLA_LIBERO_VIDEO_BS192_8K
UNIVLA_LIBERO_IMG_BS192_8K
```

注意：官方 `WORLD_MODEL_POSTTRAIN` 不是 LIBERO-only world model，而是混合数据集训练出来的。

## Video policy 和 Image policy 的通俗区别

`UNIVLA_LIBERO_IMG_BS192_8K`：

```text
看当前一张图，然后预测动作。
```

输入近似：

```text
任务指令 + 当前 static image + 当前 gripper image -> action tokens
```

`UNIVLA_LIBERO_VIDEO_BS192_8K`：

```text
看一小段过程，再预测动作。
```

当前 LIBERO video 脚本参数：

```bash
--frames 2
--action_frames 10
--video_format "interleave"
--use_gripper True
--actions True
```

输入近似：

```text
任务指令
+ 第 1 帧视觉 token + action token
+ 第 2 帧视觉 token + action token
```

通俗说：

```text
Image policy = 看照片做决定
Video policy = 看短视频做决定
```

官方 README 结果：

```text
LIBERO image SFT AVG: 94.8
LIBERO video SFT AVG: 95.5
```

## Policy post training 是否需要 LIBERO-like 数据

需要。如果只是评测官方 checkpoint，不需要训练数据；但只要进行 policy post training，就需要类似 LIBERO 的专家示范数据。

原因：

```text
policy post training 学的是：
任务指令 + 当前/历史视觉观测 -> 机器人动作 token
```

因此训练样本必须至少包含：

```python
{
    "text": instruction,
    "image": [...],          # static camera VQ tokens
    "gripper_image": [...],  # gripper camera VQ tokens, LIBERO 脚本使用
    "action": [...]          # 专家动作序列
}
```

LIBERO video policy 脚本中关键参数：

```bash
--actions True
--apply_loss_on_only_action True
--use_gripper True
--actions_format "fast"
--video_format "interleave"
```

这表示第二阶段不是继续单纯预测图像，而是在给定语言和视觉上下文后，监督学习专家动作 token。

如果目标是在 LIBERO 上评测成功率，最好使用 LIBERO demonstrations 做 policy post training，因为相机视角、任务分布、动作空间和环境动力学都与评测一致。使用其他机器人数据也可以训练出 policy，但会产生 domain/action mismatch，LIBERO 成功率通常不能直接对齐官方结果。

进一步澄清：

```text
policy post training 用 LIBERO，不只是为了获取 camera 的 vision embedding / vision token。
更关键的是需要 LIBERO 中的：

任务指令 + 相机观测 + 专家动作

这三者的配对关系。
```

在当前 UniVLA 代码中，camera 图像会先通过 Emu3-VisionVQ 提取成 `.npy` 视觉 token。policy 训练时这些视觉 token 作为输入上下文，真正的监督目标是专家动作经过 FAST tokenizer 后得到的 action token。

因此：

```text
camera/VQ token 告诉模型“现在看到什么”
action token 标签告诉模型“这种情况下应该怎么动”
```

## world model post training 配置

入口：

```text
scripts/pretrain/train_video_1node.sh
```

核心默认配置：

```bash
NGPUS=8
PRETRAIN=pretrain/Emu3-Base
MODEL_CONFIG=configs/moe_fast_video_pretrain.json
DATAPATH=datasets/post_train_data/meta/world_model_post_train_v3.pkl
max_steps=30000
per_device_train_batch_size=1
gradient_accumulation_steps=2
frames=6
action_frames=5
max_position_embeddings=6400
bf16=True
tf32=True
deepspeed=scripts/sft/zero3_offload.json
```

全局 batch size：

```text
8 GPUs * 1 per_device_batch * 2 grad_acc = 16
```

这个阶段不训练动作：

```bash
--actions False
--apply_loss_on_only_vision True
--apply_loss_on_only_action False
--post_training True
```

它学习的是：

```text
任务文本 + 视频帧 token -> 预测视觉 token
```

## world model post training 需要的资源

最低需要：

```text
UniVLA repo
Emu3-Base checkpoint
Emu3-VisionVQ / Emu3-VisionTokenizer
world model 训练 pickle
对应的 VQ token .npy 文件
```

推荐机器：

```text
8 x A100 80GB / A800 80GB / H100
```

勉强：

```text
8 x A100 40GB + ZeRO-3 offload
```

不推荐：

```text
单张 3090 / 4090
```

软件：

```text
Linux
CUDA 12.x
torch==2.4.0+cu124
transformers==4.44.0
flash-attn==2.5.7
deepspeed
```

`requirements.txt` 里没有明确列 `deepspeed`，但训练脚本使用了它，需要额外安装。

## 预计耗时

按官方风格 `30000 steps`：

```text
8 x A100/H100: 大约 1 到 4 天
8 x A800/A100 40GB + offload: 可能 3 到 7 天
更少 GPU: 时间明显变长，且可能显存不够
```

建议先 smoke test：

```bash
--max_steps 1000
```

确认数据、路径、显存和分布式训练都能跑通。

## 两个 post training 的资源和时间估计

当前仓库默认脚本对应的是：

```text
world model post training:
scripts/pretrain/train_video_1node.sh

LIBERO video policy post training:
scripts/simulator/libero/train_libero_video.sh
```

### World model post training

默认参数：

```text
max_steps = 30000
frames = 6
action_frames = 5
max_position_embeddings = 6400
per_device_train_batch_size = 1
gradient_accumulation_steps = 2
NGPUS = 8
global batch = 8 * 1 * 2 = 16
```

资源和时间估计：

```text
8 x A100/H100 80GB: 约 1 到 4 天
8 x A800/A100 40GB + ZeRO-3 offload: 约 3 到 7 天
少于 8 张大显存卡: 不建议正式训练，通常需要降 batch/frames/seq len
```

如果是 LIBERO-only world model，单步计算量仍接近默认配置；只有在减少 `max_steps` 时总时间才会明显下降。建议先用 `max_steps=1000` 做 smoke test。

### LIBERO video policy post training

默认参数：

```text
max_steps = 8000
frames = 2
action_frames = 10
max_position_embeddings = 3200
per_device_train_batch_size = 3
gradient_accumulation_steps = 8
NGPUS = 8
global batch = 8 * 3 * 8 = 192
```

资源和时间估计：

```text
8 x A100/H100 80GB: 约 8 到 24 小时
8 x A800/A100 40GB + ZeRO-3 offload: 约 1 到 3 天
4 x 80GB GPU: 可尝试，但需要调小 batch 或增加 grad_acc，时间约翻倍
单张 4090/3090: 只适合 debug，不适合正式复现
```

注意：正式耗时还包括数据准备和 VQ token 提取。LIBERO-only 的数据准备相对可控；官方式混合数据路线需要下载、处理和 VQ 化多个大数据集，前处理时间和存储压力可能比训练本身还麻烦。

## 官方披露的硬件和时间

已查阅 UniVLA 论文/arXiv HTML 与本地提取文本。

官方对 world model post training 有明确披露：

```text
observations resize: 256 x 256
input frames: 6
max sequence length: 6400
full-parameter training
50k steps
32 x NVIDIA A100 40GB
约 4-5 天
```

位置：

```text
UniVLA paper / arXiv 2506.19850
Appendix A Implementation Details
Post-training Stage
PDF 第 16 页
在 Table 8: Post-training datasets 之后的段落
原文所在句附近：
"During training, the observations are resized to 256×256, using six frames as input, with the maximum sequence length set to 6400. We perform full-parameter training for 50k steps using 32 A100 GPUs (40GB), which takes approximately 4–5 days."
```

论文正文另有一处描述 post-training stage：

```text
30K steps
batch size 64
supervision only on vision tokens
```

这和 appendix 中 `50k steps` 存在不完全一致；公开仓库脚本目前是 `max_steps=30000`，默认 `NGPUS=8`，与 appendix 的 32 张 A100 配置也不完全一致。

官方对 LIBERO/CALVIN policy finetuning 披露的是：

```text
A100 GPUs
batch size 192
8k steps
LIBERO 使用 third-person + wrist-view RGB images，均为 200 x 200
```

但论文没有明确写 LIBERO policy post training 用了多少张 A100、跑了多久。公开脚本默认 `NGPUS=8`，`per_device_train_batch_size=3`，`gradient_accumulation_steps=8`，刚好得到 global batch 192，因此按公开代码复现通常理解为 8 卡配置。

## 推荐后续路线

用户已经将后续工作明确成三条路线：

```text
路线 1：直接用官方 checkpoint 在 LIBERO 上评测
路线 2：只使用 LIBERO 数据进行两类 post training，得到自己的 checkpoint 后评测
路线 3：按官方方法融合多种数据集进行两类 post training，得到自己的 checkpoint 后评测
```

### 路线 1：官方 checkpoint 直接评测

目的：

```text
验证 LIBERO 环境、评测脚本、官方 checkpoint 下载和推理流程是否跑通。
```

需要：

```text
UNIVLA_LIBERO_VIDEO_BS192_8K 或 UNIVLA_LIBERO_IMG_BS192_8K
LIBERO evaluation 环境
```

不需要重新训练。

### 路线 2：LIBERO-only 两阶段 post training

目的：

```text
研究只用 LIBERO 数据做 world model post training + policy post training 的效果。
```

流程：

```text
Emu3-Base
-> LIBERO-only world model post training
-> 自己的 WORLD_MODEL_POSTTRAIN_LIBERO
-> LIBERO policy post training
-> 自己的 UNIVLA_LIBERO_VIDEO_LIBERO_WM
-> LIBERO evaluation
```

注意：

```text
这不是官方完全相同设置，结果可能低于官方，因为 world model 阶段缺少混合数据。
```

### 路线 3：官方式混合数据两阶段 post training

目的：

```text
尽可能复现官方训练设定。
```

流程：

```text
Emu3-Base
-> 多数据集混合 world model post training
-> 自己的 WORLD_MODEL_POSTTRAIN
-> LIBERO policy post training
-> 自己的 UNIVLA_LIBERO_VIDEO
-> LIBERO evaluation
```

需要下载和处理多种数据集，成本最高。

此前建议的最稳妥路线是先做路线 1 或使用官方 world model 进行 policy 复现：

```text
1. 下载官方 WORLD_MODEL_POSTTRAIN
2. 准备 LIBERO 数据和 VQ token
3. 跑 scripts/simulator/libero/train_libero_video.sh
4. 跑 LIBERO evaluation
5. 对齐官方 UNIVLA_LIBERO_VIDEO_BS192_8K 成功率
```

之后再做路线 2：

```text
1. 新增 tools/pickle_gen/world_model_libero.py
2. 从 libero_all_norm_aug.pkl 生成 libero_world_model.pkl
3. 用 Emu3-Base 跑 LIBERO-only world model post training
4. 用该 ckpt 跑 LIBERO video policy post training
5. 与官方 WORLD_MODEL_POSTTRAIN 路线对比
```
